# Contribution Guidelines

Thank you for your interest in contributing to Proxima! Please read the
guidelines below to ensure a smooth development experience.

## Code of conduct

Please first read the [code of conduct](CODE_OF_CONDUCT.md), and abide by it
whenever you interact with the community.

## Issues

Use [issues](https://github.com/utkarshrai2811/proxima/issues) for reporting
bugs and feature requests, and for giving context to PRs you submit. To keep the
issue tracker focused on development, use
[discussions](https://github.com/utkarshrai2811/proxima/discussions) for usage
questions and non-code-related discourse.

## Pull requests

Before submitting a pull request that introduces a new feature or significantly
changes the behavior of Proxima, please consider first opening a
[discussion](https://github.com/utkarshrai2811/proxima/discussions) or
commenting on a relevant existing issue to share what you have in mind.

## Development

Requirements: Go 1.23+ and Node 20+.

```sh
make build      # builds the admin UI and the proxima binary
go test ./...   # run the test suite
```

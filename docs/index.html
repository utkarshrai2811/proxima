Proxima is a machine-in-the-middle HTTP proxy and testing toolkit for penetration testers and bug bounty hunters. One binary, runs locally, full web UI.

Capabilities

Everything you need to inspect traffic

A modern proxy core plus the offensive tooling that used to mean juggling five apps.

◆

MITM HTTP/HTTPS proxy

Intercept and log every request and response, with a searchable proxy history and project based storage.

◫

Request intercept

Pause traffic for manual review. Edit, forward, or drop requests and responses on the fly.

⚡

Intruder style fuzzer NEW

Four attack types (Sniper, Battering Ram, Pitchfork, Cluster Bomb), built in payload lists, and live results.

⇆

WebSocket proxying NEW

Log frames in both directions, inject your own, and stream the session live.

⇪

Export NEW

Turn captured traffic into Burp Suite XML, ready to run curl commands, or an OpenAPI 3.0 skeleton.

⚙

JavaScript plugins NEW

Hook the pipeline with sandboxed scripts. Per plugin timeouts and error isolation keep the proxy alive.

🔒

Hardened by default

Optional API key auth, DNS rebinding protection, and body read limits on the admin interface.

▤

Modern web UI

A fast React and Vite interface with a dark monospace design that runs entirely on your machine.

Install

Up and running in one line

Pick your platform. Each option drops a single self contained binary on your system.

# Homebrew (Apple Silicon and Intel)
brew tap utkarshrai2811/proxima
brew trust utkarshrai2811/proxima      # one-time: Homebrew gates third-party taps
brew install --cask proxima

# Proxima is not yet notarized, so clear the macOS quarantine flag once:
xattr -d com.apple.quarantine "$(brew --prefix)/bin/proxima"

proxima

The trust and quarantine steps are one-time. Homebrew only enforces tap trust when that mode is on, and running it otherwise is harmless. The quarantine step clears the "Apple could not verify proxima" Gatekeeper warning shown for any unsigned binary; redo it after a version upgrade. 

# Scoop
scoop bucket add proxima https://github.com/utkarshrai2811/scoop-proxima
scoop install proxima

proxima
# Debian / Ubuntu
sudo dpkg -i proxima_*_linux_amd64.deb

# Fedora / RHEL / openSUSE
sudo rpm -i proxima_*_linux_amd64.rpm

# Alpine
sudo apk add --allow-untrusted proxima_*_linux_amd64.apk

Packages and tarballs are attached to every release for amd64 and arm64. 

# Requires Go 1.25+ and Node 20+
git clone https://github.com/utkarshrai2811/proxima.git
cd proxima
make build
./proxima
First run: open http://localhost:8080, point your browser proxy at 127.0.0.1:8080, then install the CA certificate from the Settings page to intercept HTTPS. The proxy traffic endpoint is never gated by authentication.

Quick start

Three steps to your first capture

1

Start Proxima

Run proxima. It listens on port 8080 and creates its data directory automatically.

2

Point your browser at it

Set your system or browser HTTP proxy to 127.0.0.1:8080, then install the CA certificate from Settings to decrypt HTTPS.

3

Browse and inspect

Open http://localhost:8080. Every request shows up in the proxy log, ready to intercept, replay, fuzz, or export.
